# BrainRush 1.17.00 업데이트 기획안

## 0. 업데이트 목표

1.17.00은 신규 플레이 모드와 프로필 능력치 개편을 중심으로 구성한다.

핵심 메시지는 다음과 같다.

> 혼자서도 정식으로 두뇌 게임을 플레이하고, 내 실력이 더 정확한 능력치로 기록된다.

이번 버전의 확정 중심축은 **혼자하기 모드**와 **능력치 계산 구조 개편**이다. **무한모드**는 아이디어와 기본 구조를 준비하되, 정식 출시 범위는 추후 확정한다.

## 1. 작업 기간

- 예상 작업 기간: 약 2주
- 목표 버전: 1.17.00
- 우선순위:
  1. 혼자하기 모드
  2. 능력치 계산용 결과 데이터 구조
  3. 프로필 육각레이더 스탯 개편
  4. 무한모드 기본 설계 또는 준비중 처리

## 2. 혼자하기 모드

### 2-1. 모드 정의

혼자하기 모드는 연습모드처럼 혼자 입장하지만, 연습이 아니라 정식 플레이에 가까운 단독 세션으로 본다.

랭크나 일반 게임처럼 상대와 매칭하지 않고, 플레이어 혼자 미니게임을 진행한다. 부담 없이 실력을 측정하고 보상을 얻는 모드로 사용한다.

### 2-2. 기본 규칙

- 입장 시 연필 1개를 소모한다.
- 한 세션은 총 3판으로 구성한다.
- 3판을 모두 완료하면 세션을 종료한다.
- 광고 노출 규칙은 일반/랭크 게임과 동일하게 구성한다.
- 두 판에 한 번 광고가 나오도록 한다.
- 매칭 상대는 없다.
- 세션 결과는 3판 누적 기준으로 보여준다.

### 2-3. 연습모드와의 차이

연습모드는 부담 없이 게임을 익히는 용도다.

혼자하기 모드는 다음 성격을 가진다.

- 연필을 소모한다.
- 정식 기록 또는 통계 반영 대상이 될 수 있다.
- 결과 화면과 누적 성과가 존재한다.
- 프로필 능력치 계산에 활용할 수 있다.

### 2-4. 구현 방향

가능하면 기존 일반/랭크 게임의 흐름을 재사용한다.

재사용 후보:

- 연필 소모 로직
- 광고 노출 타이밍
- 미니게임 라운드 진행
- 게임 결과 저장
- 결과 화면 구성
- 프로필 스탯 반영 흐름

다만 매칭, 상대 점수 동기화, 승패 판정은 혼자하기 모드에 맞게 분리하거나 비활성화한다.

## 3. 무한모드

### 3-1. 현재 상태

무한모드는 아직 구체적인 플레이 방식이 확정되지 않은 모드다.

후보 아이디어:

- 미니게임 하나를 선택해서 오래 플레이하기
- 1만점 달성 도전
- 제한 시간 없이 실수할 때까지 계속하기
- 제한 시간 안에 최고점 도전
- 특정 미니게임별 챌린지 모드

### 3-2. 1.17.00에서의 권장 범위

1.17.00에서는 무한모드를 메인 기능으로 완성하기보다, 다음 중 하나로 제한한다.

- 기본 타입과 메뉴 구조만 준비한다.
- UI에는 준비중으로 표시한다.
- 특정 미니게임 1개만 내부 테스트용으로 연다.
- 보상은 최소화하고 기록 저장 위주로 구성한다.

### 3-3. 보류 이유

무한모드는 다음 요소가 함께 결정되어야 한다.

- 난이도 상승 방식
- 보상 지급 기준
- 광고 노출 기준
- 피로도 관리
- 랭킹 연결 여부
- 미니게임별 밸런스

따라서 1.17.00에서는 혼자하기 모드와 스탯 개편을 먼저 안정화하고, 무한모드는 1.17.x 또는 이후 버전에서 확장하는 방향이 안전하다.

## 4. 프로필 능력치 개편

### 4-1. 현재 방식의 한계

현재 플레이어 능력치는 게임에서 이길 때마다 정해진 점수가 올라가고, 그 값이 프로필의 육각레이더에 표시되는 방식이다.

이 방식은 구현이 단순하지만, 실제 플레이 성향이나 실력을 충분히 반영하지 못한다.

### 4-2. 변경 목표

각 미니게임의 플레이 결과에서 반응속도, 정답률, 실수율, 처리량 같은 지표를 계산한다.

계산된 지표를 평균 또는 백분율로 변환해 프로필 육각레이더에 표시한다.

### 4-3. 추천 스탯 항목

프로필 육각레이더는 다음과 같은 공통 스탯으로 구성하는 것을 우선 검토한다.

- 반응속도: 입력 또는 정답까지 걸린 시간
- 정확도: 정답률
- 집중력: 연속 성공률, 실수 감소, 안정성
- 처리속도: 제한 시간 내 처리량
- 기억력: 기억형 미니게임 성과
- 판단력: 선택형, 규칙형, 논리형 미니게임 성과

### 4-4. 계산 원칙

- 미니게임별 원본 결과를 공통 지표로 변환한다.
- 원본 지표를 바로 프로필에 덮어쓰지 않는다.
- 최근 N판 또는 누적 평균을 사용한다.
- 한 판의 결과 때문에 능력치가 급격히 흔들리지 않게 한다.
- 게임별 난이도와 점수 분포가 다를 수 있으므로 백분율 변환을 고려한다.

### 4-5. 데이터 예시

미니게임 결과에서 수집할 수 있는 값의 예시는 다음과 같다.

- 평균 반응 시간
- 최고 반응 시간
- 정답 수
- 오답 수
- 정답률
- 콤보 수
- 최대 콤보
- 제한 시간 내 처리한 문제 수
- 게임별 최종 점수
- 게임별 난이도 또는 라운드 정보

## 5. 1.17.00 확정 추천 범위

1.17.00에서 확정 구현하는 것을 추천하는 항목은 다음과 같다.

- 혼자하기 모드 추가
- 혼자하기 3판 세션 플로우
- 혼자하기 입장 시 연필 1개 소모
- 혼자하기 광고 노출 규칙 연결
- 혼자하기 결과 화면 구성
- 혼자하기 기록 저장
- 능력치 계산용 공통 결과 데이터 구조 추가
- 일부 미니게임부터 반응속도와 정답률 기록
- 프로필 육각레이더 계산 방식 교체 또는 병행 준비

## 6. 1.17.00에서 욕심내지 않을 항목

다음 항목은 1.17.00에서 모두 완성하려고 하면 범위가 커질 수 있다.

- 모든 미니게임의 스탯 계산 완성
- 무한모드 정식 출시
- 스탯 공식의 완전한 밸런싱
- 보상 구조 전체 재설계
- 랭킹, 업적, 상점과의 완전 연동

## 7. 2주 작업 계획

### 1주차

- 혼자하기 모드 구조 구현
- 혼자하기 3판 세션 흐름 연결
- 연필 소모 로직 연결
- 광고 타이밍 연결
- 결과 처리 및 결과 화면 연결

- 스탯 수집용 공통 데이터 구조 설계

### 2주차

- 미니게임별 기본 지표 수집 시작
- 반응속도, 정답률 등 공통 지표 계산
- 프로필 육각레이더 계산 방식 교체 또는 병행
- 연필 소모, 광고 타이밍, 세션 종료 QA
- 앱 재시작 또는 중단 후 복구 흐름 점검
- 무한모드 기본 껍데기 또는 준비중 UI 추가

## 8. 구현 시 참조 원칙

앞으로 1.17.00 관련 작업을 할 때는 이 문서를 기준으로 범위를 판단한다.

특히 다음 기준을 우선한다.

- 혼자하기 모드는 1.17.00 핵심 기능으로 본다.
- 무한모드는 정식 출시보다 설계와 확장 준비를 우선한다.
- 프로필 스탯은 실제 플레이 데이터를 기반으로 계산하는 방향으로 전환한다.
- 스탯 계산은 한 번에 완성하기보다 공통 구조를 만들고 미니게임별로 점진 적용한다.
- 기존 일반/랭크/연습모드의 코드를 가능한 한 재사용한다.
- 2주 안에 끝낼 수 있는 범위를 넘기지 않도록 한다.
