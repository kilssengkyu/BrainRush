# BrainRush 개발 로드맵 및 우선순위 제안

현재 구현된 18개 미니게임과 연필 시스템을 바탕으로, 앱 출시를 위한 향후 계획을 정리했습니다.

## 🎯 우선순위 추천 (Priority)

제안해주신 기능들은 모두 "경쟁형 두뇌 게임"에 필수적인 요소들입니다. 하지만 **효율적인 개발과 출시**를 위해 다음과 같은 순서로 진행하는 것을 추천합니다.

### 1단계: 동기 부여 및 유지 (Retention Core) - **[가장 시급]**
유저가 게임을 **계속 플레이할 이유**를 만들어야 합니다. 미니게임이 30개가 되더라도, 보상이나 성취감이 없으면 금방 이탈합니다.
*   **레벨 시스템 (Leveling)**:
    *   단순 MMR(실력 점수) 외에, 플레이 횟수에 따른 절대적인 성장의 척도가 필요합니다.
    *   게임 종료 시 경험치(EXP) 획득 -> 레벨 업 -> 연필 충전 or 코스튬 보상 등으로 연계.
*   **랭크 티어 (Rank Tiers)**:
    *   점수(MMR)를 시각화 (브론즈, 실버, 골드, 플래티넘, 다이아).
    *   유저에게 명확한 목표("골드는 가야지")를 제시합니다.
*   **전체 랭킹 (Global Leaderboard)**:
    *   "나는 전체 몇 등인가?"를 보여주어 최상위권 유저들의 경쟁심을 자극합니다.

### 2단계: 모바일 앱 전환 준비 (Mobile Optimization) - **[중요]**
게임이 30개가 되기 전에, 현재 게임들이 모바일 환경(터치감, 화면 비율, 성능)에서 완벽한지 검증해야 합니다. 게임이 많아진 뒤에 수정하려면 일이 30배가 됩니다.
*   **Capacitor 적용**: 현재 React 웹앱을 iOS/Android 앱으로 패키징.
*   **터치 UX 개선**: 모바일에서는 '클릭'이 아닌 '터치'입니다. 버튼 크기, 햅틱(진동) 반응 등을 점검해야 합니다.
*   **광고 시스템 실적용**: AdMob 등 실제 수익화 모듈 연동.

### 3단계: 콘텐츠 확장 (Content Expansion)
시스템이 안정화되면 콘텐츠를 채워 넣습니다.
*   **나머지 미니게임 12종 추가**: 총 30종 달성.
*   (전략) 출시 시점에는 25개 정도로 시작하고, 출시 후 업데이트로 5개를 추가하는 것도 "지속적인 관리"를 보여주는 좋은 전략입니다.

## ⚖️ 기능 평가 및 제안

| 제안된 기능 | 평가 | 의견 |
| :--- | :--- | :--- |
| **미니게임 30종** | 필수 | 30개는 훌륭한 볼륨입니다. 단, 퀄리티 유지가 중요하며 일부는 출시 후 업데이트용으로 남겨두는 전략도 유효합니다. |
| **iOS/AOS 앱** | **핵심 목표** | 웹사이트보다는 앱이 접근성이 훨씬 좋습니다. `Capacitor`를 사용하여 현재 코드를 최대한 재사용하는 것이 효율적입니다. |
| **레벨 시스템** | **필수** | 실력이 늘지 않아도 "오래하면 오른다"는 보상이 있어야 라이트 유저가 떠나지 않습니다. |
| **랭크 티어** | **필수** | 경쟁 게임의 핵심입니다. 티어별 아이콘/테두리 등 시각적 보상이 동반되어야 합니다. |
| **전체 랭킹** | **추천** | 구현 난이도 대비 효과가 좋습니다. (일간/주간/전체 랭킹으로 세분화 추천) |

### 💡 추가 제안 시스템 (Optional but High Impact)
1.  **상세 통계 (Statistics)**:
    *   "나의 반응속도는 상위 5%", "암기력은 하위 20%" 처럼 유저의 강점/약점을 분석해주는 기능. 두뇌 트레이닝 앱으로서 매력이 크게 상승합니다.
2.  **프로필 꾸미기 (Cosmetics)**:
    *   레벨업이나 랭크 달성 보상으로 '프로필 테두리', '칭호', '아바타' 등을 주면 동기부여가 됩니다. (단순 연필 보상보다 강력)
3.  **튜토리얼 (Tutorial)**:
    *   앱 설치 후 첫 실행 시 가이드가 없으면 유저는 당황합니다. 간단한 온보딩 프로세스가 필요합니다.
